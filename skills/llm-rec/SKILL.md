## LLM Knowledge Base and Recommendation

管理两个目录：raw/（不可变的原始材料）和 wiki/（整理后的知识文章）。原始资料放入 raw/，你将其整理成 wiki 文章，wiki 会随时间不断积累。

## Architecture

三层结构，全部位于用户的项目根目录下：

raw/ — 不可变的原始材料。只读，永不修改。按主题子目录组织（例如 raw/machine-learning/）。
wiki/ — 整理后的知识文章。完全由你管理。按主题子目录组织，仅一级：wiki/<主题>/<文章>.md。包含两个特殊文件：
wiki/index.md — 全局索引。每篇文章一行，按主题分组，包含链接 + 摘要 + 更新日期。
wiki/log.md — 仅追加的操作日志。
SKILL.md（本文件）— 模式层。定义结构和工作流规则。

模板位于本文件相对的 references/ 目录中。需要原始文件、文章、归档页面或索引的确切格式时，请读取它们。

### 初始化

仅在首次摄取时触发。检查 `raw/` 和 `wiki/` 是否存在。仅创建缺失的部分；绝不覆盖现有文件：

- `raw/` 目录（含 `.gitkeep`）
- `wiki/` 目录（含 `.gitkeep`）
- `wiki/index.md` — 标题 `# 知识库索引`，正文为空
- `wiki/log.md` — 标题 `# Wiki 日志`，正文为空

若查询或检查无法找到 wiki 结构，告知用户："请先运行摄取以初始化 wiki。" 不要自动创建。

## 摄取

将来源获取到 raw/，然后编译到 wiki/。必须两步都执行，没有例外。

### 获取（raw/）

1. 使用环境提供的网络或文件工具获取来源内容。如果无法访问来源，请用户直接粘贴。

2. 选择主题目录。先检查现有的 `raw/` 子目录；如果主题足够接近，则复用。只有真正不同的主题才创建新子目录。

3. 保存为 `raw/<topic>/YYYY-MM-DD-descriptive-slug.md`。
   - Slug 来自来源标题，短横线连接，最多 60 个字符。
   - 发布日期未知 → 从文件名中省略日期前缀（例如 `descriptive-slug.md`）。元数据中的 Published 字段仍会出现；设为 `Unknown`。
   - 如果同名文件已存在，追加数字后缀（例如 `descriptive-slug-2.md`）。
   - 包含元数据头：来源 URL、收集日期、发布日期。
   - 保留原始文本。清理格式噪音。不改写观点。

   确切格式参见 `references/raw-template.md`。

### 编译（wiki/）

确定新内容归属何处：

- **与现有文章核心论点相同** → 合并到该文章。将新来源添加到 Sources/Raw。更新受影响章节。
- **新概念** → 在最相关的主题目录创建新文章。文件名以概念命名，而非原始文件。
- **跨多个主题** → 放在最相关的目录。添加 See Also 交叉引用指向其他位置的相关文章。

这些并非互斥。单个来源可能需要合并到一篇文章，同时为其引入的某个不同概念创建单独文章。所有情况下，检查事实冲突：如果新来源与现有内容矛盾，用来源归属标注分歧。合并时，在合并后的文章中注明冲突。当冲突内容位于不同文章时，在两篇文章中都注明并交叉链接。

文章格式参见 `references/article-template.md`。要点：
- Sources 字段：作者、组织或出版物名称 + 日期，分号分隔。
- Raw 字段：指向 raw/ 文件的 markdown 链接，分号分隔。
- 从 `wiki/<topic>/` 出发的相对路径使用 `../../raw/<topic>/<file>.md`（向上两级到项目根目录）。

### 级联更新

处理完主文章后，检查连锁影响：

1. 扫描同一主题目录中的文章，查找受新来源影响的内容。
2. 扫描 `wiki/index.md` 中其他主题的条目，查找涵盖相关概念的文章。
3. 更新每篇内容受到实质影响的文章。每个更新的文件刷新其 Updated 日期。

归档页面永不级联更新（它们是时间点快照）。

### 摄取后

更新 `wiki/index.md`：为每篇涉及的文章添加或更新条目。添加新主题章节时，包含一行描述。Updated 日期反映文章知识内容最后变更的时间，而非文件系统时间戳。格式参见 `references/index-template.md`。

追加到 `wiki/log.md`：

```
## [YYYY-MM-DD] ingest | <主文章标题>
- Updated: <级联更新的文章标题>
- Updated: <另一篇级联更新的文章标题>
```

无级联更新时省略 `- Updated:` 行。

## 查询

搜索 wiki 并回答问题。触发示例：
- "关于 X 我知道什么？"
- "总结与 Y 相关的所有内容"
- "基于我的 wiki 比较 A 和 B"

### 步骤

1. 读取 `wiki/index.md` 定位相关文章。
2. 读取这些文章并综合答案。
3. 优先使用 wiki 内容而非自身训练知识。用 markdown 链接引用来源：`[文章标题](wiki/topic/article.md)`（对话内引用使用项目根目录相对路径；wiki/ 文件内部，使用相对于当前文件的路径）。
4. 在对话中输出答案。除非被要求，否则不写入文件。

### 归档

当用户明确要求归档或保存答案到 wiki 时：

1. 将答案写入为新 wiki 页面。参见 `references/archive-template.md`。将对话引用转换为归档页面时，将项目根目录相对路径（例如 `wiki/topic/article.md`）重写为文件相对路径（例如 `../topic/article.md`，同目录则为 `article.md`）。
   - Sources：答案中引用的 wiki 文章的 markdown 链接。
   - 无 Raw 字段（内容不来自 raw/）。
   - 文件名反映查询主题，例如 `transformer-architectures-overview.md`。
   - 放在最相关的主题目录。
2. 始终创建新页面。绝不合并到现有文章（归档内容是综合答案，非原始材料）。
3. 更新 `wiki/index.md`。摘要前加前缀 `[Archived]`。
4. 追加到 `wiki/log.md`：
   ```
   ## [YYYY-MM-DD] query | Archived: <页面标题>
   ```

## 检查

对 wiki 进行质量检查。两类检查，权限级别不同。

### 确定性检查（自动修复）

自动修复以下问题：

**索引一致性** — 将 `wiki/index.md` 与实际 wiki/ 文件对比（排除 index.md 和 log.md）：
- 文件存在但索引缺失 → 添加条目，占位符为 `(no summary)`。Updated 使用文章元数据中的 Updated 日期（若存在）；否则回退到文件最后修改日期。
- 索引条目指向不存在的文件 → 在索引中标记为 `[MISSING]`。不删除条目；让用户决定。

**内部链接** — 对 wiki/ 文章文件中的每个 markdown 链接（正文和 Sources 元数据），排除 Raw 字段链接（由下方 Raw 引用验证）和 index.md/log.md（上方已处理）：
- 目标不存在 → 在 wiki/ 中搜索同名文件。
  - 恰好一个匹配 → 修复路径。
  - 零个或多个匹配 → 向用户报告。

**Raw 引用** — Raw 字段中的每个链接必须指向存在的 raw/ 文件：
- 目标不存在 → 在 raw/ 中搜索同名文件。
  - 恰好一个匹配 → 修复路径。
  - 零个或多个匹配 → 向用户报告。

**See Also** — 每个主题目录内：
- 添加相关文章之间明显缺失的交叉引用。
- 删除指向已删除文件的链接。

### 启发式检查（仅报告）

这些依赖你的判断。报告发现，不自动修复：

- 文章间事实矛盾
- 被新来源取代的过时主张
- 来源分歧处缺失的冲突标注
- 无其他 wiki 文章入链的孤立页面
- 缺失的跨主题引用
- 频繁提及但缺乏专门页面的概念
- 所引来源文章自归档后已大幅更新的归档页面

### 检查后

追加到 `wiki/log.md`：

```
## [YYYY-MM-DD] lint | 发现 <N> 个问题，自动修复 <M> 个
```

## 约定

- 全文使用标准 markdown 和相对链接。
- wiki/ 仅支持一级主题子目录。不嵌套更深。
- 日志条目、收集日期和归档日期使用今天日期。Updated 日期反映文章知识内容最后变更时间。Published 日期来自来源（不可用时用 `Unknown`）。
- wiki/ 文件内部，所有 markdown 链接使用相对于当前文件的路径。对话输出中，使用项目根目录相对路径（例如 `wiki/topic/article.md`）。
- 摄取更新 `wiki/index.md` 和 `wiki/log.md`。归档（来自查询）更新两者。检查更新 `wiki/log.md`（仅自动修复索引条目时更新 `wiki/index.md`）。普通查询不写入任何文件。
